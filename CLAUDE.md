## スキル

### /checkpoint
作業の区切りで呼ぶ。[DEVLOG.md](DEVLOG.md) の先頭に新しいエントリを追加する。

フォーマット:
```markdown
### YYYY-MM-DD HH:MM — <タイトル>
- **Agent**: Claude
- **やったこと**: ...
- **なぜ**: ...
- **次のタスク**: ...
- **未解決課題**: ...
```

既存エントリは改変しない。**セッション終了前に必ず実行**する。

### /resume
数日空いた再開時に呼ぶ。[DEVLOG.md](DEVLOG.md) の直近エントリを読んで状況を再構築し、「現状」「次のタスク」「未解決課題」をユーザーに要約提示する。

## 作業上のクセ

### コミット粒度
- **機能追加や修正ごとに細かくコミット**。まとめてドカンとしない
- コミットメッセージは [AGENTS.md](AGENTS.md) のコミット規約に従う

### UI 変更時
- スマホ実機で確認してもらう前提で、**具体的なテスト手順を提示**する
- 「リロードして、○○タブを開いて、○○ボタンを押してみてください」レベルの粒度

### 破壊的操作
- ブランチ削除・force push・DB データ削除等は **必ず事前確認**
- ユーザーが「任せる」と言わない限り、勝手に実行しない

### GAS 変更
- `apps_script/Code.js` を変更したら、コミット後に **clasp で即デプロイする**
- 自動では反映されない（フロントと違う）

**デプロイコマンド（1行で完結）:**
```bash
clasp push && clasp deploy --deploymentId AKfycby_nlACo4deB_VEuxTCv8uQWta7ifcf28RCaLk523UwMRqe2v4iCKrWz8ZEhwi7CUBD --description "vX.X 説明"
```
- `clasp push` → Apps Script エディタにコード反映
- `clasp deploy` → 本番URLを同じまま新バージョンとして公開
- `.clasp.json` の `scriptId` と `rootDir: "apps_script"` は設定済み・ログイン済み

### 文脈の保存
- 重要な意思決定（「○○方式を採用」「○○は不可能と判明」等）は
  作業の区切りで [DEVLOG.md](DEVLOG.md) に記録
- 後日の自分や別エージェントが同じ轍を踏まないようにする

## 過去の重要な意思決定（ダイジェスト）

詳細は [DEVLOG.md](DEVLOG.md) にあるが、特に重要な決定：

| 決定 | 経緯 |
|---|---|
| **iOS音声セッション問題は諦めた** | 複数アプローチ（AudioContext無音、silent audio loop、oscillator burst）を試したが全て無効。iOS Safari のプラットフォーム制約として受容 |
| **speechSynthesis.pause() は使わない** | killTTS に pause を入れたら TTS が全部無音になるバグに遭遇。以降 cancel のみで統一 |
| **Inbox は非同期処理方式** | quickCapture API で素投入→後で翻訳＋保存。同期的に全部処理するのは摩擦が大きい |
| **データベースは Google Sheets + GAS** | 個人用途なので認証なし「全員」アクセスで運用。無料枠で十分 |
| **4スタイル翻訳は段階的ロード** | 1回の大きなAPI呼出→結果表示まで時間かかる問題を、Normal即表示+他スタイル並行で解決（v1-bundled → v2-progressive） |
| **学習メモの視点は言語ペアで自動切替** | 日→英 なら英語（訳文）を解説、英→日 なら英語（原文）を解説。常に「日本人が学ぶ英語」視点で統一 |

## 頻出シーン別の振る舞い

### ユーザーが「こういう機能を作りたい」と相談してきた
- 実装に飛びつかず、まず **論点整理と提案** を返す
- 選択肢を示して判断を仰ぐ
- 決まったら TodoWrite で分解 → 実装

### ユーザーがバグを報告してきた
- 再現条件を確認
- 原因の **仮説を立てて共有**（決めつけない）
- 仮説が当たっていそうなら最小修正
- 関係ない部分のリファクタは混ぜない

### 長文メッセージが来た
- 要点を箇条書きで整理して返す
- 選択肢を提示して判断を仰ぐ（勝手に決めない）

## このプロジェクト特有の注意

- スマホ（iPhone）での使用が主用途。**常にモバイル UI を優先**
- ユーザーが日常で使うものなので **摩擦を極小化** する方向に倒す
- 1ファイル main app（app.js）なので巨大だが、セクション分割されている。
  変更時は該当セクションのみに手を入れる
